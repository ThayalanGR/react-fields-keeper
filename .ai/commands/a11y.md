# Command: Add Accessibility to a Component

Use when adding ARIA support or keyboard interaction to a new or existing component.

## Checklist for a new interactive element

| Element type | Required attributes |
|---|---|
| Clickable div | `role="button"`, `tabIndex={0}`, `aria-label`, `onKeyDown` (Enter + Space) |
| List container | `role="list"`, `aria-label` |
| List item | `role="listitem"`, `tabIndex={0}`, `aria-label`, `aria-disabled` |
| Collapsible section | `role="button"`, `aria-expanded={!isCollapsed}`, `aria-label="Expand/Collapse {name}"` |
| Selectable item | `aria-selected={isSelected}`, `aria-label` |
| Disabled element | `aria-disabled={true}`, `tabIndex={-1}` |
| Inline edit input | `aria-label="Edit label for {name}"`, `onKeyDown` with `e.stopPropagation()` |
| Region / landmark | `role="region"`, `aria-label` |

## Keyboard interaction patterns in this codebase

**Navigation within a list (bucket items, root items)**
```tsx
onKeyDown={(e) => {
    if ((e.target as HTMLElement).tagName === 'INPUT') return;
    if (e.key === 'ArrowDown' || e.key === 'ArrowUp') {
        e.preventDefault();
        const container = e.currentTarget.closest('.your-list-container');
        if (!container) return;
        const items = Array.from(container.querySelectorAll('[role="listitem"][tabindex="0"]')) as HTMLElement[];
        const idx = items.indexOf(e.currentTarget);
        if (e.key === 'ArrowDown' && idx < items.length - 1) items[idx + 1].focus();
        if (e.key === 'ArrowUp' && idx > 0) items[idx - 1].focus();
    }
}}
```

**Enter / Space to activate**
```tsx
onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
        e.preventDefault();
        handler();
    }
}}
```

**Inline edit — prevent key bubbling**
Always add `e.stopPropagation()` to the input's `onKeyDown` so Arrow/Delete keys don't trigger the parent item's handlers.

**Focus restoration after remove**
Use the `removeWithFocusRestore` pattern from `FieldsKeeperBucket.tsx` — find the next sibling, fall back to previous, fall back to the container.

**Focus restoration after inline edit**
Store the item's `data-field-id` before entering edit mode. After `setEditableItemId(null)`, use `requestAnimationFrame` to refocus the element via `querySelector('[data-field-id="{id}"]')`.

## Known gaps (not yet implemented)

- **Roving tabindex** — all items currently have `tabIndex={0}`. Should be one-at-a-time per WAI-ARIA composite widget spec.
- **`aria-live` announcements** — no live region for item assignment/removal events.
- **`role="listbox"` / `role="tree"`** — proper semantic structure for bucket and folder hierarchy.
- **Keyboard drag-and-drop alternative** — no pick-up/move/drop keyboard mechanism.
